# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述

### 背景與動機
許多喜歡烹飪的人常會遇到以下困擾：不知道如何管理自己手邊的私房食譜、或者看著冰箱裡剩餘的材料卻不知道能煮什麼。為了幫助這些使用者有效整理食譜並激發料理靈感，我們計畫開發一款「食譜收藏夾」系統。使用者不僅能隨時紀錄並瀏覽喜愛的食譜，更能透過現有材料進行搜尋，讓煮飯變得更輕鬆有趣。

### 目標用戶
- 愛煮飯的人
- 需要管理自己專屬食譜的人
- 常常看著冰箱裡剩餘食材不知道煮什麼的人

### 核心價值主張
- 簡單、直覺的食譜紀錄與管理。
- 以食材反向搜尋食譜，解決「今天吃什麼」的煩惱，並減少食材浪費。

---

## 2. 功能需求

### 主要功能清單
1. **新增食譜**：使用者可以自行輸入食譜名稱、材料、步驟及簡介等基本資訊。
2. **列出材料與步驟**：系統可以清楚展示每道料理所需的材料清單及圖文步驟（食譜詳情頁面）。
3. **根據材料尋找適合的食譜**：使用者可以輸入手邊的材料，系統會列出包含這些材料的相關食譜。
4. **修改與刪除食譜**：使用者可以隨時更新已新增的食譜內容，或移除不再需要的食譜。
5. **分類與標籤功能**：允許使用者替食譜加上分類或標籤（如：中式、甜點、快速料理等），方便未來快速篩選。

### 使用者故事 (User Stories)
- **新增食譜**：作為一名愛煮飯的人，我希望「能夠輕鬆記錄並新增我自己的食譜」，以便「能夠建立我的專屬料理資料庫」。
- **列出材料與步驟**：作為一名愛煮飯的人，我希望「每道食譜都能清晰列出材料與製作步驟」，以便「料理時可以按部就班地跟著做而不會出錯」。
- **根據材料尋找食譜**：作為一名愛煮飯的人，我希望「能夠輸入我擁有的剩餘食材來搜尋食譜」，以便「不用煩惱今天該煮什麼，並減少食材浪費」。
- **修改與刪除食譜**：作為一名愛煮飯的人，我希望「能夠隨時編輯或刪除已經紀錄的食譜」，以便「在需要調整配方或整理收藏清單時能保持資料庫整潔」。
- **分類與標籤功能**：作為一名愛煮飯的人，我希望「能夠將食譜設定不同的分類與標籤」，以便「未來想要找特定類型（例如甜點、快速料理）的食譜時能更快尋獲」。

## 3. 非功能需求

### 技術限制
- 原則上使用 Python 框架 **Flask** 開發後端。
- 前端網頁渲染使用 **Jinja2** 模板引擎及原生 HTML/CSS/JS 開發。
- 資料庫採用 **SQLite**，以減少設定及佈署成本。

### 效能與安全考量
- 效能：頁面載入速度需在 2 秒內，保證搜尋功能對於包含關鍵字的材料查詢有即時的反應。
- 安全：確保系統防止基礎的 SQL Injection 及 XSS 攻擊，對所有輸入欄位（材料、分類、食譜內容等）進行跳脫與過濾處理。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 提供食譜的新增功能（名稱、材料清單、製作步驟）。
- 提供食譜的列表檢視及詳細內容頁面。
- 支援透過輸入食材來進行基本的關鍵字搜尋功能。

### Should Have (應該具備)
- 已建立食譜的修改 (編輯) 與刪除功能。
- 將材料清單與步驟明確地段落化排列與展示。

### Nice to Have (最好能有)
- 食譜的分類與標籤功能。
- 支援上傳食譜封面圖片。
- 登入/註冊功能（多使用者支援與個人化隔離）。
