# PRD - 食譜收藏夾 (Recipe Collection)

## 1. 專案概述
- **背景與動機**：對於經常下廚的人來說，常會忘記曾經做過的美味料理，或者是網路上看到的食譜散落各處難以管理。本系統旨在提供一個集中管理、快速查詢與記錄細節的專屬食譜收藏平台。
- **目標用戶**：煮飯的人、家庭主夫/主婦、料理愛好者。
- **核心價值主張**：提供簡單易用的介面，讓使用者可以輕鬆記錄食譜的材料與步驟。特別強調「步驟所需等待時間」的記錄，幫助料理者在廚房能更好地掌握時間、安排多道菜的烹飪順序，提升整體效率。

## 2. 功能需求
以下為系統的主要功能及其使用者故事：

1. **新增食譜**
   - 作為 [煮飯的人]，我希望 [能夠新增包含名稱、簡介等基本資訊的食譜]，以便 [將我喜歡的料理記錄下來]。
2. **列出材料與步驟**
   - 作為 [煮飯的人]，我希望 [能夠詳細記錄並檢視每道料理的所需材料清單和逐步操作流程]，以便 [在料理時能按部就班地進行準備與烹調]。
3. **步驟所需等待時間記錄**
   - 作為 [煮飯的人]，我希望 [能在各個步驟中特別標註所需的等待時間（例如：醃製 30 分鐘、燉煮 1 小時）]，以便 [在等待期間能安排煮其他菜或做其他家事，不浪費時間]。
4. **編輯與刪除食譜**
   - 作為 [煮飯的人]，我希望 [能夠修改已收藏的食譜內容或刪除不再需要的食譜]，以便 [隨時根據心得調整配方，並保持食譜庫的整潔]。
5. **食譜搜尋與分類**
   - 作為 [煮飯的人]，我希望 [能透過關鍵字搜尋或分類標籤（如：台式、甜點、快速料理）來尋找食譜]，以便 [在不知道要做什麼菜時，能快速從我的收藏夾中找到靈感]。

## 3. 非功能需求
- **技術限制**：系統須使用 Flask + Jinja2 作為網頁框架，並搭配 SQLite 作為資料庫儲存食譜資料。
- **效能考量**：頁面載入時間需小於 2 秒，確保在廚房使用手機或平板檢視時能有流暢的體驗。
- **安全考量**：需防範常見的 Web 攻擊（如 SQL Injection, XSS），並對使用者輸入的表單內容進行基礎驗證。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增食譜（包含名稱與簡介）。
  - 新增並列出該食譜的材料清單。
  - 新增並列出該食譜的步驟，且能記錄與顯示步驟所需等待時間。
  - 檢視所有食譜列表（首頁），以及單一食譜的詳細頁面。
- **Should Have (應該有)**：
  - 編輯與刪除現有食譜。
  - 食譜分類標籤。
- **Nice to Have (可以有)**：
  - 烹飪計時器（針對步驟中的等待時間提供倒數計時功能）。
  - 關鍵字搜尋食譜。
  - 圖片上傳功能。
