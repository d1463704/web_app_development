# PRD – 個人記帳簿系統（學生版）

## 1. 專案概述

### 背景與動機
學生族群往往收入有限（生活費、兼職薪資），卻缺乏系統化的方式追蹤每日花費，導致月底常常「不知道錢去哪了」。本系統提供簡單直觀的記帳介面，讓學生能快速記錄收入與支出，並透過統計圖表了解自己的財務狀況，養成良好的理財習慣。

### 目標用戶
- **主要用戶**：在學學生（高中、大學、研究生）
- **次要用戶**：剛出社會的社會新鮮人，收入尚不穩定者

### 核心價值主張
提供一個**零負擔、快速上手**的記帳工具，讓學生能在 30 秒內完成一筆記帳，並在月底自動生成消費報告，幫助養成良好的財務管理習慣。

---

## 2. 功能需求

| 功能 | 使用者故事 |
|------|------------|
| **收入記錄** | 作為 **學生**，我希望能 **記錄每筆收入來源（生活費、獎學金、打工）**，以便 **清楚掌握本月總收入**。 |
| **支出記錄** | 作為 **學生**，我希望能 **快速記錄每筆消費，並加上分類（餐飲、交通、娛樂）**，以便 **了解金錢流向**。 |
| **餘額統計** | 作為 **學生**，我希望能 **即時查看本月剩餘金額（收入 - 支出）**，以便 **判斷是否需要節制消費**。 |
| **分類報表** | 作為 **學生**，我希望能 **用圓餅圖或長條圖查看各分類支出比例**，以便 **找出最大花費項目**。 |
| **月度預算設定** | 作為 **學生**，我希望能 **為每個月設定總預算或各分類預算**，以便 **在快超出預算時收到提醒**。 |
| **帳戶管理** | 作為 **學生**，我希望能 **管理不同資金來源（現金、悠遊卡、行動支付）**，以便 **分開追蹤各錢包的餘額**。 |
| **資料匯出** | 作為 **學生**，我希望能 **將記帳資料匯出為 CSV**，以便 **自行用 Excel 進一步分析**。 |

---

## 3. 非功能需求

- **技術限制**：使用 Python Flask + Jinja2 作為後端框架，SQLite 作為本地資料庫，CSS/Bootstrap 5 處理前端樣式。
- **效能**：頁面載入與操作回應時間 ≤ 500ms（本地開發環境）。
- **易用性**：新增一筆記帳操作步驟不超過 3 步，介面簡潔，適合非技術使用者。
- **安全與隱私**：所有資料僅存於使用者本機 SQLite 檔案，不上傳至任何雲端服務。
- **可維護性**：遵循 Flask MVC 架構，程式碼符合 PEP8 規範，方便未來擴充。

## 4. MVP 範圍

| 等級 | 功能 |
|------|------|
| **Must Have（一定要有）** | 收入記錄、支出記錄、本月餘額統計、基本分類管理、帳戶管理 |
| **Should Have（應該要有）** | 月度預算設定與提醒、支出分類圓餅圖、資料匯出 CSV |
| **Nice to Have（有更好）** | 每週消費報告、多月份趨勢折線圖、行動版響應式介面優化 |
